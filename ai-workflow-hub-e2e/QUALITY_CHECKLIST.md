# 上线前质量检查清单

> 每个项目接入 TestFrame 后，在上线前必须逐项检查。

---

## 一、测试覆盖检查

### 1.1 Android App

- [ ] 核心用户路径（登录→首页→购买→支付）有 Maestro 冒烟测试
- [ ] 主要功能模块有 Airtest/Poco 回归测试
- [ ] 冒烟测试在 < 10 分钟内完成
- [ ] 回归测试在 < 30 分钟内完成
- [ ] 所有测试脚本在 CI 环境中可执行
- [ ] 图像识别的基准截图已更新到当前版本

### 1.2 微信小程序

- [ ] 核心用户路径有自动化测试
- [ ] 微信开发者工具自动化模式已配置
- [ ] CI 构建机已预装开发者工具
- [ ] wx API Mock 已覆盖支付/登录等关键场景
- [ ] 小程序各页面可被测试脚本正常访问

### 1.3 H5/uni-app

- [ ] 核心页面有 Playwright 测试
- [ ] 覆盖 Chromium + WebKit（模拟 iOS Safari）
- [ ] 移动端视口模拟已配置
- [ ] 微信 JS-SDK 相关功能有 Mock
- [ ] 网络 Mock 覆盖了所有后端依赖

### 1.4 后端 API

- [ ] 所有公开 API 有 MeterSphere 测试用例
- [ ] 核心业务流程有场景测试（多接口串联）
- [ ] 正常路径 + 异常路径（401/403/404/500）已覆盖
- [ ] 敏感接口有鉴权检查
- [ ] 性能基线已建立

---

## 二、云真机兼容性

- [ ] 已选择至少 top20 国产安卓机型
- [ ] 兼容性测试可在 CI 中自动触发
- [ ] 已知的兼容性问题已记录（不同OS版本/厂商定制）
- [ ] WeTest API Key 已配置且可用
- [ ] 兼容性报告可与 Allure 报告关联

---

## 三、崩溃监控

- [ ] Sentry SDK 已集成到 Android App
- [ ] Sentry SDK 已集成到 Web/H5
- [ ] Bugly SDK 已集成到 Android App
- [ ] Bugly 已集成到微信小程序
- [ ] 崩溃率 < 0.5%
- [ ] 已知崩溃都有修复计划或豁免标记
- [ ] Release 追踪已配置（崩溃可关联到代码版本）

---

## 四、报告与归因

- [ ] Allure 报告能聚合所有工具的测试结果
- [ ] 失败用例有截图/视频附件
- [ ] 报告链接自动发送到团队IM
- [ ] 归因规则引擎能匹配常见失败模式
- [ ] 未匹配的失败有人工分析流程
- [ ] 趋势数据可追溯（至少保留30天）

---

## 五、CI/CD 门禁

- [ ] PR 创建时自动执行冒烟测试
- [ ] main 分支合并后自动执行回归测试
- [ ] Release Tag 触发全量测试（含兼容性）
- [ ] 每日定时质量巡检已配置
- [ ] 门禁通过率阈值已设定（冒烟100%，回归95%）
- [ ] 失败通知已配置（IM/邮件/Sentry告警）
- [ ] 门禁 FAIL 时阻断合并/发布
- [ ] 有紧急绕过流程（需审批）

---

## 六、环境与配置

- [ ] 所有敏感信息通过环境变量注入
- [ ] `.env` 文件已加入 .gitignore，不在仓库中
- [ ] CI 环境变量（Secrets）已配置
- [ ] 测试环境与生产环境隔离
- [ ] 测试账号池与真实用户数据隔离
- [ ] 第三方服务 API Key 有有效期监控
- [ ] 配置文件模板（.yaml）与实际配置（.local.yaml）分离

---

## 七、性能基线

- [ ] 冒烟测试执行时间基线已建立
- [ ] 回归测试执行时间基线已建立
- [ ] Android App 冷启动时间已记录
- [ ] H5 首页加载时间（FCP/LCP）已记录
- [ ] API 平均响应时间已记录
- [ ] 内存/CPU 消耗基线已建立
- [ ] 性能回退检测阈值已设定

---

## 八、安全合规

- [ ] 测试代码不包含生产密钥
- [ ] Mock 数据不使用真实用户数据
- [ ] 崩溃日志不包含用户敏感信息（PII）
- [ ] 截图/视频中的敏感信息已脱敏
- [ ] 第三方工具的数据存储位置已确认
- [ ] 最小权限原则：API Key 只授予必要权限

---

## 九、文档与培训

- [ ] SETUP.md 中所有人工步骤已验证可执行
- [ ] README.md 快速开始指南已验证
- [ ] 各工具的 access token / API key 获取步骤已记录
- [ ] 常见问题（FAQ）已整理
- [ ] 团队成员至少1人会触发手动测试
- [ ] 团队成员至少1人会查看和解读Allure报告

---

## 十、上线决策

在以上检查项全部通过后，进行最终上线决策：

- [ ] 冒烟测试 100% 通过
- [ ] 回归测试 ≥ 95% 通过
- [ ] 兼容性测试 ≥ 90% 通过
- [ ] 零 P0 级崩溃
- [ ] 零严重安全漏洞
- [ ] 性能无回退（与上版本基线对比）
- [ ] 所有门禁 PASS

**上线决策**：
- [ ] ✅ 批准上线
- [ ] ⚠ 条件批准（注明条件：________）
- [ ] ❌ 禁止上线（注明原因：________）
